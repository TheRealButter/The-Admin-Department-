# Client Admin Working System v1

This folder turns Client Admin from a demo into a testable operating system.

It defines:

- required fields
- onboarding status gates
- document checklist checks
- agreement/payment checks
- folder and handover checks
- calculated flags
- weekly report calculations
- 10-record test data
- pass/fail expectations

## Working-system rule

The tracker must be able to show:

- missing required documents
- agreement not signed
- payment not confirmed
- folder not created or folder link missing
- internal handover incomplete
- active/ready status blocked
- ownerless active records
- overdue next actions
- weekly report counts

If those cannot be calculated from the tracker and checklist, the system is not ready for a paid pilot.
