# Client Admin Pilot Setup Checklist

Use this checklist for the first paid Client Admin pilot.

## Pilot scope

One client. One onboarding workflow. One document checklist. One weekly report.

Do not include complex project management, legal document drafting, payment automation, or full CRM migration in the first pilot.

## Access needed

- [ ] Last 5–10 recent new clients or sample onboarding cases
- [ ] Current welcome message/email
- [ ] Current required document list
- [ ] Current agreement/payment process
- [ ] Current folder structure
- [ ] Current internal handover steps
- [ ] Service packages offered
- [ ] Client onboarding owner names
- [ ] Weekly reporting preference

## Setup steps

- [ ] Copy Client Admin tracker template
- [ ] Add client-specific service packages
- [ ] Add required documents by package
- [ ] Add onboarding statuses
- [ ] Add owners/assignees
- [ ] Add folder naming rules
- [ ] Add welcome message template
- [ ] Add missing-document follow-up template
- [ ] Add internal handover checklist
- [ ] Add weekly report format
- [ ] Add 5–10 real sample onboarding cases
- [ ] Mark status, owner, missing items, and next action

## Client handover

- [ ] Explain onboarding statuses
- [ ] Explain document checklist
- [ ] Explain folder structure
- [ ] Explain agreement/payment tracking
- [ ] Explain internal handover rules
- [ ] Explain weekly report
- [ ] Confirm who updates the tracker daily
- [ ] Confirm first review date

## Pilot success criteria

- All sample clients are visible in one tracker.
- Every onboarding case has a status.
- Every missing document has an owner and follow-up date.
- Every Ready to Start client has folder/agreement/payment status checked.
- Client understands the weekly report.
- Client can see value without needing full project management software.

## Expansion after pilot

Only after the workflow works:

- onboarding form
- folder creation automation
- email draft automation
- missing-document reminders
- weekly report automation
- Supabase/Admin HQ dashboard
