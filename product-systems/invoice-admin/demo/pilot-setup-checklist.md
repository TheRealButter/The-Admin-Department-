# Invoice Admin Pilot Setup Checklist

Use this checklist for the first paid pilot.

## Pilot scope

One client. One invoice workflow. One approval process. One weekly report.

Do not include accounting integrations, payment automation, or complex multi-department setup in the first pilot.

## Access needed

- [ ] Sample supplier invoice emails or exported PDFs
- [ ] Current invoice tracker/spreadsheet if any
- [ ] Current supplier list if any
- [ ] Current folder structure
- [ ] Approval owner names
- [ ] Approval thresholds
- [ ] Document naming preference
- [ ] Weekly reporting preference

## Setup steps

- [ ] Copy invoice tracker template
- [ ] Create supplier register
- [ ] Create exception queue
- [ ] Create folder structure
- [ ] Add 5–10 real sample invoices
- [ ] Capture required fields
- [ ] Apply statuses
- [ ] Flag missing info and duplicates
- [ ] Add approval owner and next action
- [ ] Prepare first weekly report

## Client handover

- [ ] Explain statuses
- [ ] Explain exception queue
- [ ] Explain approval rules
- [ ] Explain folder naming
- [ ] Explain weekly report
- [ ] Confirm who updates the tracker
- [ ] Confirm first review date

## Pilot success criteria

- All sample invoices are visible in one tracker.
- Every invoice has a status.
- Every exception has an owner and next action.
- Client understands the weekly report.
- Client can see value without needing full automation.

## Expansion after pilot

Only after the workflow works:

- Gmail label intake
- attachment saving
- OCR/extraction
- duplicate checks
- weekly report automation
- Supabase/Admin HQ dashboard
