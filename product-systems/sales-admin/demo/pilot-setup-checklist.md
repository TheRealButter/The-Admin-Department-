# Sales Admin Pilot Setup Checklist

Use this checklist for the first paid Sales Admin pilot.

## Pilot scope

One client. One enquiry workflow. One follow-up process. One weekly report.

Do not include complex CRM migration, paid ads management, or autonomous message sending in the first pilot.

## Access needed

- [ ] Last 10–20 recent enquiries
- [ ] Current lead spreadsheet or notes if any
- [ ] Current WhatsApp/email/form sources
- [ ] Current quote process
- [ ] Current follow-up process
- [ ] Main services sold
- [ ] Lead owner names
- [ ] Quote approval owner
- [ ] Weekly reporting preference

## Setup steps

- [ ] Copy Sales Admin tracker template
- [ ] Add client-specific lead sources
- [ ] Add services/categories
- [ ] Add statuses
- [ ] Add lead owners
- [ ] Add urgency rules
- [ ] Add quote follow-up rules
- [ ] Add message templates
- [ ] Add weekly report format
- [ ] Add 10–20 real sample leads
- [ ] Mark status, owner, next action, and follow-up due date

## Client handover

- [ ] Explain lead statuses
- [ ] Explain first-response rule
- [ ] Explain quote follow-up rule
- [ ] Explain lost/cold lead rules
- [ ] Explain weekly report
- [ ] Confirm who updates the tracker daily
- [ ] Confirm first review date

## Pilot success criteria

- All sample leads are visible in one tracker.
- Every active lead has a status.
- Every Quote Sent lead has a follow-up due date.
- Every urgent lead has an owner.
- Client understands the weekly report.
- Client can see value without needing a full CRM.

## Expansion after pilot

Only after the workflow works:

- website form intake
- Gmail lead intake
- WhatsApp click-to-send links
- reminder automation
- missed-response alerts
- Supabase/Admin HQ dashboard
